# Historia de JavaScript

## 1995

### Orígenes: 
A principios de los años 90, el navegador web Mosaic ganó popularidad y se convirtió en el primer navegador ampliamente utilizado que permitía la visualización de imágenes en línea. Marc Andreessen, uno de los creadores de Mosaic, dejó la Universidad de Illinois para fundar Netscape Communications Corporation en 1994.

### Necesidad de interactividad: 
Netscape necesitaba agregar interactividad a sus páginas web para mejorar la experiencia del usuario y competir con otros navegadores emergentes. Por lo tanto, se propuso crear un lenguaje de scripting para lograr este objetivo.

### Brendan Eich: 
A finales de 1994, Brendan Eich, un programador y desarrollador talentoso, fue contratado por Netscape para trabajar en el navegador. En solo diez días, Brendan desarrolló el primer prototipo de lo que se convertiría en JavaScript.

### Mocha y LiveScript: 
El lenguaje original creado por Brendan se llamó "Mocha". Sin embargo, debido a que la compañía había llegado a un acuerdo con Sun Microsystems para licenciar el término "Java" para uso en tecnologías relacionadas, Netscape decidió cambiar el nombre del lenguaje a "LiveScript" en septiembre de 1995 para capitalizar la popularidad de Java en ese momento.

### Acuerdo con Sun Microsystems: 
El cambio de nombre a "LiveScript" llevó a cierta confusión, ya que algunos lo asociaban erróneamente con el lenguaje de programación Java. Con el objetivo de aprovechar el éxito de Java, Netscape y Sun Microsystems acordaron cambiar el nombre a "JavaScript" en diciembre de 1995. Sin embargo, es importante mencionar que JavaScript y Java son dos lenguajes de programación completamente diferentes.

### Lanzamiento: 
La primera versión pública de JavaScript se lanzó con Netscape Navigator 2.0 en diciembre de 1995. La adopción fue rápida, y JavaScript permitió a los desarrolladores web agregar interactividad a sus sitios de manera fácil y eficiente.

### Estándar ECMA: 
Dado el éxito y la creciente popularidad de JavaScript, Netscape buscó estandarizar el lenguaje para garantizar su interoperabilidad en diferentes plataformas y navegadores. En noviembre de 1996, Netscape entregó JavaScript a Ecma International para desarrollar un estándar, y este proceso llevó a la creación del estándar ECMA-262, que definió cómo debía funcionar JavaScript. A partir de entonces, JavaScript se ha mantenido y evolucionado como un lenguaje estándar y abierto.

### Competencia y evolución: 
A medida que JavaScript se establecía como un lenguaje esencial para el desarrollo web, otros navegadores, como Microsoft Internet Explorer, también implementaron su propia versión de JavaScript, llamada "JScript". Esto llevó a ciertas incompatibilidades entre los navegadores y dio lugar a la necesidad de prácticas de desarrollo específicas para abordar estos problemas.

## 1996

### Estándar ECMA: 
A fines de 1995, Netscape entregó el lenguaje JavaScript a Ecma International para desarrollar un estándar que permitiera su adopción más amplia y garantizara su interoperabilidad en diferentes plataformas y navegadores. Este proceso culminó con la creación del primer estándar oficial de JavaScript: ECMAScript, basado en el nombre de la organización Ecma International.

### ECMAScript 1 (ES1): 
En junio de 1997, Ecma International publicó la primera edición del estándar ECMAScript, formalmente conocido como ECMA-262. Esta especificación describía la funcionalidad básica de JavaScript y establecía la base para su evolución futura. ECMAScript 1 definió elementos esenciales del lenguaje, como tipos de datos, operadores, estructuras de control y funciones.

### JScript de Microsoft: 
Microsoft también estaba interesado en el desarrollo de lenguajes de scripting para navegadores y había creado su propia versión llamada "JScript". Aunque JScript estaba muy influenciado por JavaScript, había algunas diferencias y peculiaridades en la implementación.

### Guerra de navegadores: 
Durante 1996, se produjo una competencia intensa entre los navegadores web, con Microsoft Internet Explorer y Netscape Navigator como principales protagonistas. Ambos navegadores intentaron implementar sus propias versiones de JavaScript y JScript, lo que llevó a ciertas incompatibilidades y desafíos para los desarrolladores web. Esto creó lo que se conoció como la "Guerra de navegadores", en la que cada empresa buscaba superar a la otra con nuevas características y tecnologías.

### JScript se convierte en ECMA-262: 
Con el fin de abordar las incompatibilidades entre JavaScript y JScript, Microsoft se unió a Ecma International en noviembre de 1996 y entregó su especificación de JScript. Como resultado, ECMA-262 se convirtió en el estándar que cubría tanto JavaScript como JScript, lo que permitía una mayor coherencia entre los navegadores.

### Netscape Navigator 3.0: 
En agosto de 1996, Netscape lanzó Netscape Navigator 3.0, que incluía la versión mejorada de JavaScript. Esta versión presentó nuevas características y mejoras, lo que consolidó aún más el papel de JavaScript como el lenguaje de scripting principal para la web.

### Adopción generalizada: 
A medida que JavaScript se estandarizaba y mejoraba su compatibilidad entre navegadores, su adopción se generalizó rápidamente entre los desarrolladores web. Esto llevó a una explosión en la creación de sitios web más interactivos y dinámicos, sentando las bases para el desarrollo ### web moderno.

## 1997

### ECMAScript 2 (ES2): 
En junio de 1998, se lanzó la segunda edición del estándar ECMAScript (ECMAScript 2). Esta revisión trajo mejoras y correcciones a la especificación del lenguaje, lo que ayudó a estandarizar aún más el comportamiento de JavaScript en diferentes entornos y navegadores.

### Competencia entre navegadores: 
La "Guerra de navegadores" continuó en la década de 1990, con Microsoft Internet Explorer y Netscape Navigator compitiendo por dominar el mercado. A medida que ambas compañías mejoraban sus navegadores, también se centraban en mejorar el soporte para JavaScript y otras tecnologías web.

### DHTML y CSS: 
A finales de los 90, el término "DHTML" (Dynamic HTML) se popularizó y se convirtió en una forma común de describir cómo JavaScript, HTML y CSS trabajaban juntos para crear interactividad en las páginas web. JavaScript fue crucial para manipular el DOM (Document Object Model) y permitir efectos dinámicos y transiciones en las páginas web.

### Uso extendido: 
A medida que los desarrolladores se familiarizaban con JavaScript, comenzaron a utilizarlo para una variedad más amplia de funciones en los sitios web, desde validación de formularios hasta animaciones y control de eventos.

### Creación de frameworks y bibliotecas: 
A finales de los 90, empezaron a surgir los primeros frameworks y bibliotecas de JavaScript, como jQuery, que facilitaron y estandarizaron la manipulación del DOM y simplificaron el desarrollo web en múltiples navegadores.

## 1999

### ECMAScript 3 (ES3): 
En diciembre de 1999, se publicó la tercera edición del estándar ECMAScript (ECMAScript 3). Esta revisión significativa del estándar introdujo numerosas mejoras y nuevas características al lenguaje, lo que permitió un desarrollo más estructurado y avanzado en JavaScript.

### Adopción generalizada: 
A medida que JavaScript se consolidaba como un lenguaje de programación esencial para el desarrollo web, su adopción continuó creciendo a lo largo del año 1999. Los desarrolladores aprovecharon cada vez más su potencial para crear sitios web interactivos y dinámicos.

### Frameworks y bibliotecas: 
Durante la década de 1990, surgieron más frameworks y bibliotecas de JavaScript que facilitaron el desarrollo web y permitieron la creación de aplicaciones más sofisticadas. Algunos ejemplos notables de esta época son Prototype.js y Script.aculo.us, que proporcionaron herramientas para mejorar la experiencia del usuario en los navegadores.

### La evolución del DOM: 
En la década de 1990, el Document Object Model (DOM) también evolucionó para proporcionar una interfaz más sofisticada para que JavaScript pudiera interactuar con el contenido de las páginas web. Esto permitió a los desarrolladores manipular y cambiar el contenido y la estructura de las páginas de manera más eficiente.

## 2000

### ECMAScript 3 (ES3) consolidado: 
Aunque la especificación ECMAScript 3 fue publicada a finales de 1999, durante el año 2000, esta versión se consolidó y se convirtió en la base de implementación para la mayoría de los navegadores web. ECMAScript 3 trajo mejoras significativas al lenguaje y ayudó a establecer una mayor uniformidad en el comportamiento de JavaScript en diferentes entornos.

### Navegadores y adopción: 
Durante el año 2000, la guerra de navegadores continuó, con Internet Explorer de Microsoft y Netscape Navigator compitiendo por la supremacía en el mercado. Ambos navegadores continuaron mejorando su soporte para JavaScript y ECMA-262, lo que permitió a los desarrolladores crear experiencias web más interactivas y dinámicas.

### Introducción de XMLHttpRequest: 
En 2000, Microsoft presentó por primera vez el objeto XMLHttpRequest, una API que permitía a las páginas web realizar solicitudes HTTP asíncronas. Esta tecnología fue un avance crucial para el desarrollo de aplicaciones web más sofisticadas y dio lugar al concepto de "AJAX" (Asynchronous JavaScript and XML), que revolucionaría la forma en que se construían aplicaciones web interactivas.

### Desarrollo de frameworks y bibliotecas: 
Durante el año 2000, continuó el desarrollo de frameworks y bibliotecas de JavaScript para facilitar el desarrollo web. Aunque las bibliotecas como Prototype.js y Script.aculo.us surgieron en la década de 1990, su uso y popularidad crecieron en el nuevo milenio.

### JavaScript en el lado del servidor: 
A finales del año 2000, Ryan Dahl creó "Node.js", que permitía ejecutar JavaScript en el lado del servidor, en lugar de limitarlo solo al navegador. Esta innovación allanó el camino para el desarrollo de aplicaciones web en tiempo real y escalables, y Node.js se convertiría en una de las plataformas más populares para el desarrollo de servidores web y aplicaciones.

ECMAScript 3, que fue publicado a finales de 1999, se consolidó durante el año 2001 como el estándar principal de JavaScript. El uso de la tecnología AJAX (Asynchronous JavaScript and XML), que había sido introducida por Microsoft en 2000 con el objeto XMLHttpRequest, se popularizó para actualizar partes específicas de las páginas sin necesidad de recargar la página completa.

### Frameworks y bibliotecas: 
Durante el año 2001, se desarrollaron y mejoraron diversos frameworks y bibliotecas de JavaScript. Algunos ejemplos notables son Script.aculo.us y MochiKit, que facilitaban la creación de efectos visuales y animaciones en páginas web, y Prototype.js, que ofrecía utilidades y mejoras en el manejo del DOM.

### Adopción de JavaScript en el lado del servidor: 
Node.js, una plataforma que permitía ejecutar JavaScript en el lado del servidor, continuó su desarrollo y ganó popularidad en el año 2001. Esto abrió nuevas posibilidades para el desarrollo de aplicaciones en tiempo real y en el lado del servidor, lo que llevó a un crecimiento significativo en la adopción de Node.js en los años siguientes.

### Estándar DOM Nivel 2: 
Durante el año 2001, el World Wide Web Consortium (W3C) publicó el estándar DOM Nivel 2, que proporcionó una API más avanzada y consistente para interactuar con el DOM en JavaScript. Esto facilitó el desarrollo web y la manipulación de elementos de página de manera más eficiente.

### Cambios en la dinámica laboral: 
La pandemia de COVID-19 tuvo un impacto significativo en la dinámica laboral, lo que llevó a un aumento en el trabajo remoto. Los desarrolladores de JavaScript se adaptaron a estas nuevas condiciones y continuaron colaborando y compartiendo conocimientos a través de plataformas en línea y herramientas de desarrollo colaborativas.

### Adopción generalizada y comunidad activa: 
En 2020, JavaScript continuó siendo uno de los lenguajes de programación más populares y ampliamente utilizados. La evolución de ECMAScript, la consolidación de frameworks y bibliotecas modernas, y la comunidad activa de desarrolladores contribuyeron a su adopción generalizada en el desarrollo web y más allá.
